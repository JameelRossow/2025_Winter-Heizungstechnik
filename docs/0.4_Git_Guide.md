# Git Guide

## Ziel
Das Projekt soll moeglichst simpel mit Git gefuehrt werden: nur ein Branch, keine Feature-Branches, nur Commit und am Ende Tagging.

## Commit-Richtlinien
- Commit-Messages beginnen immer damit, worum es geht: entweder "Doku: ..." fuer Dokumentationsaenderungen oder "Code: ..." fuer technische Anpassungen.
- Jeder Commit fasst eine abgeschlossene Aenderung zusammen (kein halbfertiger Zustand).
- Bei Bedarf Inhalt kurz beschreiben, z. B. `Doku: Kapitel 3.2 erweitern` oder `Code: Berechnung der Heizlast angepasst`.

## Tagging
- Tags werden nur dann gesetzt, wenn wir einen Meilenstein oder eine Version fertig haben.
- Der erste Tag ist `0.1.0` (entspricht dem aktuellen Stand); neue Tags vergebt ihr, sobald eine neue Version veroeffentlicht wird.

## Workflow (lokal)
1. `git status` pruefen, um weitere Aenderungen einzusehen.
2. `git add <Datei>` (oder `git add .` bei mehreren Dateien).
3. `git commit -m "Doku: ..."` oder `git commit -m "Code: ..."` entsprechend dem Inhalt.
4. Falls alles bereit ist, nach GitHub pushen (`git push origin main` oder wie der zentrale Branch heisst).
5. Zum Abschluss ein Tag setzen, z. B. `git tag 0.1.0` und anschließend `git push origin 0.1.0`.

## Hinweise
- Keine Branches oder Merge-Strategien noetig, solange wir alle auf `main` arbeiten.
- Dokumentation und Codeaenderungen sollten sauber voneinander getrennt beschrieben werden, damit man in der Historie nachvollziehen kann, was geaendert wurde.

## Remote & GitHub
1. Ein neues GitHub-Repo legst du ueber die Website an, gibst ihm einen Namen, waehlst die Sichtbarkeit und erzeugst keine README/License, damit unsere lokale Historie nicht ueberschrieben wird.
2. Kopiere die Remote-URL (HTTPS oder SSH) und verknuepfe sie lokal mit `git remote add origin <url>`.
3. Beim ersten Push (`git push -u origin main`) wirst du zur Authentifizierung aufgefordert (Personal Access Token fuer HTTPS oder SSH-Key + Passphrase); es gibt keinen `git login`, sondern nur diesen Auth-Dialog pro Remote.
4. Danach reicht `git push origin main` fuer Änderungen und `git push origin 0.1.0` fuer den aktuellen Tag.
5. Die Repo-Erstellung passiert auf GitHub; im Terminal hängst du dich dann einfach ueber die Remote an, ohne irgendwo 'einzuloggen'.
