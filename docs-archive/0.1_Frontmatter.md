# Frontmatter

Dieses Kapitel sammelt alle Vorgaben für den Frontmatter-Block oben in jeder Markdown-Datei. Alle konfigurierbaren Werte leben direkt in der Datei; es gibt keine externe `doc_config.yaml`.

## Automatische Felder

- **id** und **title** werden beim Start von `python run.py` oder über den Livereload-Sync automatisch aus dem Dateinamen (`X.Y_Beschreibung.md`) berechnet. Diese Felder bleiben im Frontmatter zwar sichtbar, dürfen aber nie manuell editiert werden.
- Der Sync geschieht bei jedem Start von `run.py` und bei Änderungen unter `docs/` (Livereload Watcher). Sobald ein File umbenannt wird, schreibt der Sync die neuen Werte zurück in Datei und Manifest.

## Pflichtfelder

- **status**: leer lassen oder Werte wie `not_started`, `in_progress`, `done`. Wird in `docs/doc_manifest.json` und damit auf der Übersichtsseite angezeigt.
- **layout**: momentan `A4`. Weitere Layouts können später eingeführt werden, Standard bleibt `A4`.
- **force_new_page_before**: `true` erzwingt einen Seitenumbruch vor dem Kapitel (gilt für Viewer/PDF). Setze `false`, wenn das Kapitel direkt anschließen soll.
- **page_break_after**: analog zum Feld davor, aber für den Abschluss eines Kapitels.
- **visible_in_viewer**: steuert, ob Kapitel in der Viewer-Navigation und Liste landen (`false` blendet aus, `true` = default = zeigt an).

## Optionale Felder

- **description**: Freitext für die Home-Übersicht (kurzer Teaser, was im Kapitel steht).
- Weitere projektspezifische Flags können unterhalb des Standardblocks eingefügt werden. Der Sync übernimmt nur `status`, `layout`, Pagebreak-Flags und `visible_in_viewer`; alle anderen Felder bleiben unverändert.

## Workflow-Hinweise

1. Neue Datei mit leerem Frontmatter anlegen (Felder gem. Template oben).
2. `python run.py` starten (oder laufenden Livereload weiterlaufen lassen). Der Sync aktualisiert automatisch `id`, `title` und das Manifest.
3. Im Viewer prüfen: nur die Frontmatter-Werte steuern Auflistung, Reihenfolge folgt aus `id`.

Damit bleibt der Frontmatter die einzige Konfigurationsquelle; keine externe YAML wird mehr geladen.
