# User-Guide

Dieser Guide beschreibt, welche Dateien was steuern, wie der Viewer formatiert und welche Markdown-Tricks wir nutzen.

## 1. Dateien & Aufgaben
- `docs/`: Enthält ein Kapitel pro Markdown-Datei (`8.0_Anhang.md` etc.). Der Frontmatter-Block oben regelt status/layout/page-breaks/visible; `id`/`title` werden vom Dateinamen abgeleitet.
- `doc_manifest.json`: Aktuelle Liste aller Kapitel, wird von `run.py` automatisch erzeugt.
- `viewer/`: Viewer-HTML/CSS/JS, legt Century Gothic auf 12pt fest, steuert A4-Boxen, KaTeX-Rendering und die Kapitelnavigation.
- `Runbook/00_Projekteinrichtung/`: Planung, Styleguide, DokuEnv mit Workflow-Erklärung.
- `run.py`: Synchronisiert Frontmatter, erzeugt Manifest und startet optional `livereload`.

## 2. Markdown-Regeln
- **Frontmatter**: Wird automatisch befüllt, außer `status` kannst du hier zusätzliche Flags ergänzen. Niemals `id`/`title` manuell setzen.
- **Typografie**: `#`, `##`, `###` folgen den CSS-Werten im Viewer (`viewer/viewer.css`).
- **Listen**: Etwas wie `- Punkt` oder `1. Nummer`.
- **Tabellen**: Alle Einheiten in Kopfzeile; keine inline CSS.
- **Formeln**: `$...$` oder `$$...$$`.
- **Meta-Hinweise**: Schreibe kursiv oder als Zitat (`> Hinweis:`).

## 3. Workflow
1. Kapitel markdown bearbeiten.
2. `.md` speichern → `run.py` synchronisiert Frontmatter/Manifest.
3. Im Browser `http://localhost:3000/viewer/` öffnen, Viewer zeigt A4-karten.
4. Für PDF: Browserdruck (Strg+P). `@media print` regelt die Print-Ansicht.

## 4. Server beenden
- `run.py` speichert nach dem Start seine PID in `run_id.json`; das Skript `stop.py` nutzt diese Datei, um den Server kontrolliert zu beenden.
- Wenn `Strg+C` nicht reagiert, im Projektverzeichnis `python stop.py` ausführen. Das Skript prüft, ob die PID noch läuft, beendet `run.py` (ggf. über `taskkill`) und entfernt die Laufzeitdatei.
- Meldet `stop.py`, dass keine PID gefunden wurde, ist entweder kein Server aktiv oder die Datei wurde bereits entfernt.
- Falls du doch manuell eingreifen willst, hilft `Get-Process python | Format-Table Id, Path`, die passende ID mit `Stop-Process -Id <ID> -Force` oder `taskkill /PID <ID> /T /F` zu beenden und anschließend `run_id.json` zu löschen.
